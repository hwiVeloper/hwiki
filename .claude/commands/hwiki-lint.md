# hwiki Lint

위키의 건강 상태를 점검하고 문제를 보고한다.

## 사용법

```
/hwiki-lint
/hwiki-lint --fix
```

- 기본: 점검만 하고 보고서를 출력한다.
- `--fix`: 점검 후 발견된 문제를 자동으로 수정한다.

## 워크플로우

1. `$ARGUMENTS`에서 `--fix` 플래그 여부를 확인한다.
2. `wiki/` 아래 모든 마크다운 파일을 스캔한다.
3. 다음 항목을 점검한다:
   - **모순**: 페이지 간 모순되는 정보
   - **고아 페이지**: 다른 페이지에서 `[[wikilink]]`로 링크되지 않는 페이지
   - **깨진 링크**: `[[wikilink]]`가 가리키는 페이지가 존재하지 않는 경우
   - **누락된 교차참조**: 관련 있지만 서로 링크되지 않은 페이지들
   - **누락된 페이지**: 여러 곳에서 언급되지만 자체 페이지가 없는 개념/엔티티
   - **오래된 정보**: 최신 자료와 충돌하는 오래된 내용
   - **index.md 동기화**: 위키 페이지가 있는데 index.md에 없거나 그 반대
4. 점검 결과를 카테고리별로 보고한다.

### `--fix` 모드

5. 발견된 문제 중 자동 수정 가능한 것을 보여주고 수정한다:
   - 누락된 교차참조 추가
   - index.md 동기화
   - 깨진 링크 표시
6. 모순이나 오래된 정보 등 판단이 필요한 문제는 사용자에게 확인을 구한다.
7. `wiki/log.md`에 `## [YYYY-MM-DD] lint | 수정 내용 요약` 형식으로 기록한다.
8. git commit한다. 메시지: `lint: [수정 내용 요약]`

### 점검만 모드 (기본)

5. 보고서만 출력하고 파일을 수정하지 않는다.
