## 요약

## 핵심 내용

### RAG의 치명적 한계
- **매번 처음부터 다시 읽는다**: 질문할 때마다 원본 문서에서 조각을 찾아 답변을 생성하며, 지식의 누적이 발생하지 않음
- **연결성을 이해하지 못한다**: 문서 간의 모순, 과거 데이터와 최신 데이터의 충돌을 파악하지 못하고 파편화된 정보만 제공
- **탐색이 곧 휘발이다**: 아무리 훌륭한 답변과 인사이트를 도출해도, 채팅창을 닫는 순간 지식은 시스템에서 영원히 사라짐

### 핵심 전환: Retrieve → Compile
AI는 문서를 인덱싱하는 것에 그치지 않고, 문서를 읽고, 요약하고, 기존 지식망과 교차 참조하여 영구적인 마크다운 위키로 사전 컴파일한다.

> "AI가 지루한 유지보수를 전담하고, 인간은 아이디어를 탐구한다."

### RAG vs LLM Wiki 비교

| Feature | RAG | LLM Wiki |
|---|---|---|
| 데이터 처리 | 질문 시점에 원문 검색 | 사전에 문서를 위키로 컴파일 |
| 토큰 효율성 | 매번 방대한 원문 참조 (비효율) | 정제된 요약본과 인덱스만 참조 (고효율) |
| 정보의 연결성 | 단방향 (문서에서 답변으로) | 양방향 (자동 백링크 및 동적 교차 참조) |
| 지식의 축적 곡선 | 수평선 (매번 휘발) | J커브 (지속적인 복리 축적) |

### 3-Tier 아키텍처

1. **Tier 1. Raw Sources (진실의 원천)**: 웹 클리핑, PDF 등 날것의 자료. AI는 오직 읽기만 하며 절대 수정하지 않는 불변의 데이터
2. **Tier 2. Wiki (컴파일된 지식망)**: AI가 100% 소유하고 유지보수하는 마크다운 파일 디렉토리. 엔티티, 개념, 요약이 풍부히 연결된 상태
3. **Tier 3. Schema (시스템의 두뇌)**: AI의 동작 규칙, 폴더 구조, 링크 컨벤션을 통제하는 단 하나의 시스템 프롬프트 파일 (CLAUDE.md)

### 3대 핵심 Operation

1. **Ingest (지식 컴파일)**: 원본 문서 분석 → 소스 요약 페이지 생성 → 관련 엔티티/개념 페이지 10~15개 동시 업데이트 → 전체 목차 및 작업 로그 갱신
2. **Query (지식 합성 및 반환)**: 무거운 원문 검색 대신 가벼운 목차 파일을 우선 스캔하여 필요한 위키 페이지만 타겟팅. 탐색 결과가 새로운 위키 페이지로 영구 저장
3. **Lint (건강 검진 및 유지보수)**: 모순 탐지, 고아 문서 복구, 빈 공간 식별, 오래된 정보 갱신

### The Secret Sauce: CLAUDE.md
수만 줄의 RAG 데이터 파이프라인 코드를 대체하는 단 하나의 파일. AI에게 일반적인 챗봇이 아닌 체계적인 위키 사서로서의 페르소나와 강제 규칙을 부여한다.

> "AI에게 도서관의 분류 체계 자체를 학습시키는 가장 우아한 방법."

### 스케일링 전략
초기에는 목차(index.md) 스캔만으로 충분하지만, 규모가 커지면 컨텍스트 윈도우 한계에 부딪힘. 해결책으로 Vector Embeddings + BM25 + Cross-encoder Reranking을 MCP 서버로 연결하여 로컬 머신에서 50ms 이내 검색을 실현한다.

### 활용 유스케이스
- **기술 블로그 & 논문 아카이브**: 분산된 기술 문서를 통합하여 나만의 기술 백과사전 구축
- **프로젝트 회고 & 에러 트래킹**: 에러 로그와 함께 인제스트하여 동일 패턴의 오류를 기억하고 해결책 제시
- **1인 기업/팀 위키 대체**: 산발적 메시지, 회의록, API 스펙을 던져주면 AI가 자동으로 구조화된 팀 위키로 변환
- **이력서 & 경력 기술서 자동화**: 진행 중인 프로젝트 일지와 메모를 쌓아두면, 연결된 AI가 모든 맥락을 엮어 완벽한 포트폴리오로 합산

### 핵심 메시지
지식의 가치보다 기하급수적으로 늘어나는 유지보수 비용이 개인 위키/노트 정리를 포기하게 만들었다. LLM은 지루해하지 않고, 실수하지 않으며, 한 번에 수십 개의 문서를 연결한다. **Bookkeeping Cost = 0** — 마침내 유지보수 비용이 0에 수렴하는 지식 관리의 시대가 열렸다.

## 5분 퀵스타트
1. Obsidian 새 볼트 생성 및 스키마 세팅 (`mkdir raw wiki`, CLAUDE.md 작성)
2. Web Clipper로 데이터 수집 (Ingest) — `raw/` 폴더에 저장
3. 터미널에서 `> claude` 실행 — AI 정원사가 위키를 구축하는 동안 커피를 마신다
